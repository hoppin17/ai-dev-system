# AI_RULES.md

## 목적

이 문서는 AI 도구(GPT, Cursor, Codex 등)를 사용하여 개발 및 작업을 수행할 때
일관된 작업 방식과 품질을 유지하기 위한 기본 규칙을 정의한다.

AI는 **설계 → 구현 → 검토**의 협업 도구이며,
AI의 출력은 항상 검토와 판단을 거쳐 사용한다.

---

# 1. AI 역할 정의

AI는 다음 역할을 수행한다.

1. 시스템 설계 보조
2. 코드 작성 보조
3. 코드 리뷰
4. 문서 정리
5. 문제 해결 아이디어 제안

AI는 **최종 의사결정자가 아니다.**
모든 결정은 사용자가 한다.

---

# 2. AI 도구별 역할

## GPT (설계 및 검토)

GPT는 다음 작업에 사용한다.

* 아키텍처 설계
* 시스템 구조 설계
* 작업 계획 수립
* 코드 리뷰
* 문제 해결 전략 제안
* 문서 정리
* handoff 문서 생성

GPT는 **코드 작성보다는 설계와 검토 중심으로 사용한다.**

---

## Cursor (코드 작성)

Cursor는 다음 작업에 사용한다.

* 실제 코드 작성
* 코드 수정
* 리팩토링
* 테스트 코드 작성

Cursor에게는 **명확한 지시사항과 작업 범위를 전달한다.**

---

# 3. 작업 절차 (Workflow)

모든 작업은 다음 절차를 따른다.

1️⃣ 문제 정의
2️⃣ GPT를 통해 설계 및 작업 계획 수립
3️⃣ 작업 계획 문서 작성
4️⃣ Cursor에게 구현 지시
5️⃣ 구현 결과 검토
6️⃣ GPT를 통해 코드 리뷰
7️⃣ 필요 시 수정

---

# 4. 계획 기반 개발 원칙

AI에게 바로 코드를 작성하게 하지 않는다.

반드시 다음 순서를 따른다.

1. 문제 정의
2. 설계
3. 구현
4. 검토

AI는 항상 **계획 기반(plan-driven)**으로 사용한다.

---

# 5. 코드 생성 규칙

AI가 작성하는 코드는 다음 기준을 따른다.

* 가독성 우선
* 단순한 구조
* 명확한 함수 분리
* 불필요한 복잡성 금지

다음은 금지한다.

* 과도한 추상화
* 불필요한 디자인 패턴
* 이해하기 어려운 코드

코드작성시 다음과 같은 규칙도 따른다

*규칙 1*

루트에는 실행 진입점만 둔다
-run.py
-scheduler_gui.py
-README.md
루트에 잡파일 쌓이지 않게.

*규칙 2*

실험 파일은 무조건 experiments/
“나중에 지울까?” 고민하지 말고 일단 거기로.

*규칙 3*

죽은 파일은 archive/
삭제 결정 못하겠으면 거기로.

*규칙 4*
어떤 프로젝트에도도 README 1장으로 현재 구조 설명.

---

# 6. 코드 수정 규칙

AI에게 코드 수정을 요청할 때는 다음을 명확히 한다.

* 수정 목적
* 변경 범위
* 변경 이유

AI는 **기존 코드를 불필요하게 변경하지 않는다.**

---

# 7. 문서화 규칙

모든 중요한 작업은 문서로 기록한다.

문서 종류

* README
* Handoff
* Architecture 문서
* 운영 문서

AI는 문서 작성을 보조한다.

---

# 8. Handoff 규칙

대화가 길어지거나 작업 단위가 끝나면
다음 문서를 생성한다.

1. Conversation Handoff
2. Project Handoff

Handoff는 다음 대화나 작업에서
context를 유지하기 위한 목적이다.

---

# 9. 보안 규칙

AI에 입력하지 않는 정보

* API keys
* 비밀 토큰
* 민감한 개인정보
* 내부 보안 정보

필요한 경우 데이터를 익명화하여 사용한다.

---

# 10. 품질 원칙

AI 출력은 항상 다음 기준으로 검토한다.

* 논리적 일관성
* 코드 품질
* 단순성
* 유지보수 가능성

AI의 결과를 **무조건 신뢰하지 않는다.**

---

# 11. 지속적 개선

AI 사용 방식은 지속적으로 개선한다.

필요한 경우 다음을 업데이트한다.

* AI_RULES.md
* Workflow
* Prompt templates
* Handoff format
